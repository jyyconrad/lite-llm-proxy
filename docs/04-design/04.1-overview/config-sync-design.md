# 配置同步模块设计文档

**最后更新**：2026-03-20
**版本**：v1.0.0

---

## 概述

配置同步模块负责在应用启动时同步 `litellm_config.yaml` 配置文件与数据库中的模型配置，确保配置的一致性。

### 设计目标

1. **数据库优先**：数据库配置始终优先于 YAML 配置
2. **增量同步**：只同步新增模型，不覆盖已有配置
3. **自动检测**：自动检测配置文件和数据库的变更
4. **幂等性**：多次执行同步操作结果一致

---

## 同步策略

### 数据库优先原则

```
┌─────────────────────────────────────────────────────────┐
│                    配置变更检测                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   YAML 变更？ ────是────> 只添加新模型到数据库              │
│      │                                                  │
│      └───否──> 数据库变更？ ──是──> 保持数据库版本         │
│                         │                               │
│                         └───否──> 跳过同步               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 详细规则

| 场景 | YAML 变更 | 数据库变更 | 处理策略 |
|------|----------|-----------|----------|
| 首次启动 | - | - | 从 YAML 同步所有模型到数据库 |
| YAML 新增模型 | 是 | 否 | 只添加新模型到数据库 |
| YAML 修改现有模型 | 是 | 否 | 忽略，保持数据库版本 |
| 数据库独立变更 | 否 | 是 | 保持数据库版本 |
| 无变更 | 否 | 否 | 跳过同步 |

## 同步流程

### 启动时同步流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      sync_on_startup()                           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
              ┌───────────────────────────────┐
              │   1. 计算 YAML 配置文件哈希      │
              │      yaml_hash = SHA256(content) │
              └───────────────────────────────┘
                              │
                              ▼
              ┌───────────────────────────────┐
              │   2. 获取数据库配置并计算哈希    │
              │      db_hash = SHA256(configs)  │
              └───────────────────────────────┘
                              │
                              ▼
              ┌───────────────────────────────┐
              │   3. 获取检查点记录             │
              │      checkpoint = get()         │
              └───────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              │                               │
              ▼                               ▼
     ┌─────────────────┐            ┌─────────────────┐
     │ checkpoint=None │            │ checkpoint 存在  │
     │ (首次启动)       │            │               │
     └─────────────────┘            └─────────────────┘
              │                               │
              ▼                               ▼
    ┌───────────────────┐           ┌───────────────────────┐
    │ 执行全量同步        │           │ 比较哈希检测变更       │
    │ sync_yaml_to_db() │           │ yaml_changed = ...    │
    └───────────────────┘           │ db_changed = ...      │
                                    └───────────────────────┘
                                              │
                        ┌─────────────────────┼─────────────────────┐
                        │                     │                     │
                        ▼                     ▼                     ▼
              ┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
              │ yaml_changed=T  │   │ db_changed=T    │   │ 均无变更         │
              │ (YAML 有更新)    │   │ (数据库有更新)   │   │               │
              └─────────────────┘   └─────────────────┘   └─────────────────┘
                        │                     │                     │
                        ▼                     ▼                     ▼
              ┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
              │ 只添加新模型      │   │ 保持数据库版本   │   │ 跳过同步         │
              │ 更新检查点       │   │ 更新检查点       │   │               │
              └─────────────────┘   └─────────────────┘   └─────────────────┘
```

### 哈希对比逻辑

```python
# 检查 YAML 是否有更新
yaml_changed = yaml_hash != checkpoint.yaml_hash

# 检查数据库是否有更新
db_changed = db_hash != checkpoint.db_hash

# 判断同步状态
is_synced = not yaml_changed and not db_changed
```

## 关键设计决策

### 1. 为什么数据库优先？

**问题**：YAML 和数据库配置不一致时，以哪个为准？

**决策**：数据库优先

**原因**：
- 数据库配置可以通过 API 动态修改，更灵活
- 生产环境可能已有手动调整的配置
- 避免 YAML 意外覆盖生产配置

### 2. 为什么只添加新模型？

**问题**：YAML 修改了现有模型参数，是否同步？

**决策**：不同步，只添加新模型

**原因**：
- 保持数据库优先原则
- 避免意外覆盖生产配置
- 需要修改时通过 API 操作

### 3. 为什么使用哈希对比？

**问题**：如何高效检测配置变更？

**决策**：SHA256 哈希对比

**原因**：
- 高效：O(1) 对比 vs O(n) 逐字段对比
- 准确：内容变更必然导致哈希变化
- 成熟：Git 等系统广泛使用
