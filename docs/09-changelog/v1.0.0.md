## 新增功能

### 配置同步功能

新增配置同步模块，支持 `litellm_config.yaml` 配置文件与数据库配置之间的自动同步。

#### 核心特性

- **数据库优先策略**：数据库配置始终优先，避免 YAML 意外覆盖生产配置
- **增量同步**：只添加新模型，不更新已有模型配置
- **自动变更检测**：通过 SHA256 哈希对比自动检测 YAML 和数据库的变更
- **检查点机制**：记录同步状态，支持应用重启后断点续同步

#### 新增文件

**服务层**：
- `gateway/services/config_sync_service.py` - 配置同步服务主类

**数据库表**：
- `config_checkpoints` - 配置同步检查点表

**API 端点**（`/admin/`）：
- `GET /admin/config/sync-status` - 获取同步状态
- `POST /admin/config/sync` - 手动触发同步

#### 使用示例

**启动时自动同步**：
```python
# 应用启动时自动执行
# gateway/app.py
from gateway.services.config_sync_service import get_config_sync_service

@app.on_event("startup")
async def startup_event():
    # ... 其他初始化代码
    sync_service = get_config_sync_service()
    async with get_db_session() as db:
        result = await sync_service.sync_on_startup(db)
        logger.info(f"配置同步结果：{result}")
```

**API 查询同步状态**：
```bash
curl -X GET "http://localhost:8000/admin/config/sync-status" \
  -H "Authorization: Bearer sk-abc12345"
```

**API 手动触发同步**：
```bash
curl -X POST "http://localhost:8000/admin/config/sync" \
  -H "Authorization: Bearer sk-abc12345"
```

### 数据库变更

#### 新增表：config_checkpoints

记录配置同步检查点的表结构：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | VARCHAR(36) | 主键 UUID |
| `config_type` | VARCHAR(50) | 配置类型（当前仅支持 `litellm_config`） |
| `yaml_hash` | VARCHAR(64) | YAML 文件 SHA256 哈希 |
| `db_hash` | VARCHAR(64) | 数据库配置 SHA256 哈希 |
| `last_sync_source` | VARCHAR(20) | 最后同步来源 |
| `last_sync_time` | TIMESTAMP | 最后同步时间 |
| `yaml_updated_at` | TIMESTAMP | YAML 文件修改时间 |
| `db_updated_at` | TIMESTAMP | 数据库更新时间 |

## 技术实现

### ConfigSyncService 类

| 方法 | 职责 |
|------|------|
| `compute_yaml_hash()` | 计算 YAML 配置文件哈希 |
| `compute_db_hash()` | 计算数据库配置哈希 |
| `get_checkpoint()` | 获取检查点记录 |
| `save_checkpoint()` | 保存检查点记录 |
| `get_db_model_configs()` | 获取数据库模型配置 |
| `sync_yaml_to_db()` | 执行 YAML 到数据库的同步 |
| `sync_on_startup()` | 应用启动时同步入口 |
| `get_sync_status()` | 获取当前同步状态 |

### 哈希算法

- **YAML 哈希**：对文件完整内容进行 SHA256 哈希
- **数据库哈希**：对所有激活模型配置标准化后 JSON 序列化的 SHA256 哈希

### 同步策略

```
┌─────────────────────────────────────────┐
│            配置变更检测                   │
├─────────────────────────────────────────┤
│ YAML 变更？ → 是 → 只添加新模型到数据库    │
│   │                                     │
│   └─ 否 → 数据库变更？ → 是 → 保持数据库  │
│                           │             │
│                           └─ 否 → 跳过   │
└─────────────────────────────────────────┘
```
