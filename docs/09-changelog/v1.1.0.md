## Bug 修复

### 模型配置 404 问题修复

修复了通过 Web 界面添加的模型在调用 `/v1/chat/completions` 端点时返回 404 错误的问题。

#### 问题描述

**现象**：
- 用户通过前端 Web 界面添加模型配置
- 调用 `/v1/chat/completions` 端点时返回 `404 Model not found`

**根因**：
- `config_manager.py` 中的 `async_get_model_config` 函数从数据库读取配置后，实例化 `ModelEndPoint` 时缺少默认值处理
- 前端保存的配置可能缺少某些可选字段（如 `dimensions`）
- `ModelEndPoint` 验证失败抛出异常，导致 `async_get_model_config` 返回 `None`
- `llm.py` 检测到 `cfg is None` 后返回 404

#### 修复内容

**文件**: `config_manager.py`

1. **修复 `async_get_model_config` 函数**（第 345-354 行）:
   ```python
   # 修改前：直接通过字典键访问，缺失字段会报错
   params = ModelEndPoint(
       model=litellm_params["model"],
       api_key=litellm_params["api_key"],
       # ...
   )

   # 修改后：使用 .get() 方法提供默认值
   params = ModelEndPoint(
       model=litellm_params.get("model", ""),
       api_key=litellm_params.get("api_key", ""),
       base_url=litellm_params.get("base_url", ""),
       provider=litellm_params.get("provider", "openai"),
       max_tokens=litellm_params.get("max_tokens", 4096),
       rpm=litellm_params.get("rpm", 60),
       tpm=litellm_params.get("tpm", 100000),
       weight=litellm_params.get("weight", 1),
   )
   ```

2. **修复 `yaml_to_model_config` 方法**（第 249-272 行）:
   - 同样使用 `.get()` 方法处理单节点和多节点模式下的字段缺失问题

**文件**: `data/model_info.py`

3. **修复 `warnings` 导入问题**:
   - 第 12 行添加 `import warnings`
   - 删除第 142 行 `model_validator` 中的重复 `import warnings`

#### 默认值清单

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `model` | `""` | 模型名称（必填，空字符串用于容错） |
| `api_key` | `""` | API 密钥（必填，空字符串用于容错） |
| `base_url` | `""` | 基础 URL（可选） |
| `provider` | `"openai"` | 提供商（默认 openai） |
| `max_tokens` | `4096` | 最大令牌数 |
| `rpm` | `60` | 每分钟请求数限制 |
| `tpm` | `100000` | 每分钟令牌数限制 |
| `weight` | `1` | 负载均衡权重 |

## 代码审查

### 审查结果

**审查范围**:
- 检查 `config_manager.py` 中其他字典展开模式
- 验证错误处理逻辑
- 安全性检查

**发现**:
- `convert_legacy_model_dict` 方法中使用 `model_dict["model_name"]`，但该字段为必需字段，调用前已验证，安全风险低
